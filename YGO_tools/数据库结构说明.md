# 数据库结构说明

## 概述

工作区包含两个 SQLite 数据库文件，采用 **游戏王 YGOPro 格式** 的卡牌数据库结构。

| 文件 | 说明 |
|------|------|
| `cards.cdb` | 卡牌主数据库，存储卡牌数据与文本信息 |
| `pack.db` | 卡包数据库，存储卡牌所属卡包及稀有度信息 |

---

## 1. cards.cdb

包含 **2 张表**：

### 1.1 datas 表 — 卡牌数据

存储卡牌的数值型属性。

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER (PK) | 卡牌唯一 ID（密码） |
| `ot` | INTEGER | 所属区域（0=OCG, 1=TCG, 2=OCG+TCG, 3=动画/漫画） |
| `alias` | INTEGER | 替代卡 ID（用于同一张卡的不同版本） |
| `setcode` | INTEGER | 系列/字段编码（位掩码） |
| `type` | INTEGER | 卡牌类型（位掩码，见下方说明） |
| `atk` | INTEGER | 攻击力 |
| `def` | INTEGER | 防御力 |
| `level` | INTEGER | 等级/阶级/连接标记（低 8 位=等级，高位=标记数量等） |
| `race` | INTEGER | 种族编码 |
| `attribute` | INTEGER | 属性编码 |
| `category` | INTEGER | 效果类别（位掩码） |

### 1.2 texts 表 — 卡牌文本

存储卡牌的名称、效果描述等文本信息。

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER (PK) | 卡牌 ID，与 `datas.id` 关联 |
| `name` | TEXT | 卡牌名称 |
| `desc` | TEXT | 卡牌效果描述 |
| `str1` ~ `str16` | TEXT | 额外文本字段（如灵摆效果、素材要求等） |

## 2. pack.db

包含 **1 张表**：

### 2.1 pack 表 — 卡包信息

存储每张卡所属的卡包和稀有度信息。

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER (PK) | 自增主键 |
| `pack_id` | TEXT | 卡包编号 |
| `pack` | TEXT | 卡包名称 |
| `rarity` | TEXT | 稀有度 |
| `date` | TEXT | 日期 |

---

## 3. 表关系

```
datas.id ──── texts.id   (1:1 关系，通过 id 关联)
datas.id ──── pack.id    (1:N 关系，一张卡可出现在多个卡包中)
```

## 4. 使用示例

### 查询卡牌名称与攻击力

```sql
SELECT t.name, d.atk, d.def, d.level
FROM datas d
JOIN texts t ON d.id = t.id
WHERE t.name LIKE '%青眼%'
LIMIT 10;
```

### 查询某张卡的稀有度

```sql
SELECT t.name, p.pack, p.rarity
FROM texts t
JOIN pack p ON t.id = p.id
WHERE t.name = '青眼白龙';
```

### 查询魔法卡

```sql
SELECT t.name, d.type
FROM datas d
JOIN texts t ON d.id = t.id
WHERE (d.type & 0x2) != 0  -- type 含魔法卡标志
LIMIT 20;
```
